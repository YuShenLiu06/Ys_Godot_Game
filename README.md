# 像素弹雨 (Pixel Bullet Rain) v0.5.0

![游戏图标](icon.svg)

一个使用Godot引擎开发的2D射击生存游戏，玩家需要控制角色射击不断出现的敌人，通过击败敌人获取经验值并升级，每次升级可以选择不同的能力提升。

## 🌟 v0.5.0 版本特性

### 🎮 核心玩法
- **快节奏射击玩法**：使用WASD键移动，角色静止时自动射击
- **动态难度系统**：随着时间推移，敌人生成速度逐渐加快，血量增加
- **升级与技能选择**：通过击败敌人获取经验值，升级时从三个随机技能中选择一个
- **终极天赋系统**：1级时选择终极天赋，大幅改变游戏玩法
- **轮回系统**：10级时选择轮回，重置基础属性但保留终极效果

### 🎴 卡牌系统
- **模块化卡牌架构**：基于SOLID原则的可扩展卡牌系统
- **多种卡牌类型**：基础强化、终极天赋、轮回选项
- **标签分类系统**：支持卡牌分组和动态启用
- **智能卡牌生成**：根据游戏进度和选择动态调整卡牌池

### 👾 敌人系统
- **多种敌人类型**：普通史莱姆、巨型史莱姆、普通兔子、精英兔子
- **敌人工厂模式**：统一的敌人生成和管理系统
- **动态难度调整**：敌人血量根据玩家等级动态增长
- **权重生成系统**：基于权重的随机敌人生成机制

### 🔧 技术特性
- **信号总线系统**：松耦合的组件通信机制
- **调试控制台**：游戏内调试和测试工具
- **配置管理**：音量和游戏设置的持久化存储
- **键盘管理**：上下文感知的输入管理系统

## 🎮 游戏操作

### 基础控制
- **移动**：WASD键控制角色移动
- **射击**：角色静止时自动射击，移动时停止射击
- **转向**：左键向左转，右键向右转
- **暂停**：ESC键暂停/恢复游戏

### 调试功能（F1开启控制台）
- **经验管理**：`/exp add <amount>` - 增加经验值
- **等级设置**：`/level set <amount>` - 设置等级
- **子弹切换**：`/bullet model set <type>` - 切换子弹类型
- **终极技能**：`/ultimate <type> add <amount>` - 增加终极技能次数
- **帮助信息**：`/help` - 显示所有可用指令

## 🎯 终极天赋系统

### 1. 终极追踪子弹
- **自动索敌**：子弹自动追踪最近的敌人
- **持续攻击**：无需精确瞄准，专注躲避
- **强化选项**：转向速度提升、生存时间延长

### 2. 终极爆炸连锁
- **连锁爆炸**：击中敌人时触发范围爆炸
- **高爆发伤害**：可以一次性清除大量敌人
- **强化选项**：爆炸伤害提升、触发概率增加

### 3. 终极穿透
- **子弹穿透**：子弹可以穿透多个敌人
- **直线清理**：适合清理直线排列的敌人
- **强化选项**：穿透伤害提升、触发概率增加

## 🎴 卡牌系统详解

### 基础卡牌 (Basic)
- **子弹伤害提升**：子弹伤害×1.5，伤害>2时触发爆炸
- **开火速率提升**：开火时间×0.75，提升射击频率
- **经验获取提升**：经验获取×2.0，加速升级进程

### 强化卡牌（根据选择的终极天赋）
- **追踪子弹强化**：转向速度、生存时间提升
- **爆炸连锁强化**：爆炸伤害、触发概率提升
- **穿透强化**：穿透伤害、触发概率提升

### 轮回选项（10级）
- **轮回保留**：重置基础属性但保留终极效果
- **轮回重置**：完全重置所有属性，重新开始

## 👾 敌人图鉴

### 基础敌人
- **普通史莱姆**：血量2.0，经验3点，移动速度50像素/秒
- **巨型史莱姆**：高血量敌人，5级解锁
- **普通兔子**：快速移动敌人，5级解锁
- **精英兔子**：高血量快速敌人，5级解锁，可生成小兔子

### 敌人成长机制
- **血量公式**：`基础血量^(1.1^玩家等级)`
- **生成间隔**：随时间递减，限制在0.2-3秒之间
- **生成位置**：随机从屏幕左右两侧生成

## 📊 游戏数值

### 玩家属性
- **移动速度**：70像素/秒
- **初始等级**：1级
- **升级公式**：`ceil(10 × (1.5^当前等级))`
- **经验系数**：初始1.0，可通过卡牌提升

### 武器系统
- **普通子弹**：速度300像素/秒，生存时间3秒
- **追踪子弹**：速度200像素/秒，转向速度3.0，生存时间2.0秒
- **爆炸触发**：子弹伤害>2时触发爆炸
- **爆炸范围**：`1.1^(1 + (伤害-2)/3)`，限制在1.0-5.0之间

## 🏗️ 项目架构

### 核心系统
1. **游戏管理器** ([`Game_manager.gd`](Scripts/manager/Game_manager.gd))
   - 控制敌人生成和难度递增
   - 管理分数、经验和等级系统
   - 处理升级时的技能选择界面
   - 实现屏幕震动效果的全局接口

2. **信号总线** ([`SignalBus.gd`](Scripts/manager/SignalBus.gd))
   - 实现松耦合的组件通信
   - 管理所有游戏信号
   - 支持跨模块的状态同步

3. **卡牌系统** ([`BaseCard.gd`](Scripts/selections/BaseCard.gd))
   - 模块化卡牌架构
   - 标签分类和动态加载
   - 工厂模式创建和管理

4. **敌人工厂** ([`enemy_factory.gd`](Scripts/enemy/enemy_factory.gd))
   - 统一的敌人生成和管理
   - 权重随机生成系统
   - 动态启用/禁用敌人类型

### 设计模式
- **信号系统**：使用Godot的信号系统实现松耦合的组件通信
- **工厂模式**：用于动态创建敌人和卡牌实例
- **单例模式**：游戏管理器作为全局状态管理器
- **继承与多态**：通过基类实现不同类型对象的统一管理

## 🔧 开发环境

- **引擎版本**：Godot 4.5
- **渲染器**：Forward Plus
- **编程语言**：GDScript
- **架构原则**：SOLID原则，KISS原则，DRY原则

## 🎮 游戏策略

### 新手推荐路线
1. **初期选择**：根据游戏风格选择合适的终极天赋
2. **前期发展**：优先选择基础强化卡牌
3. **中期强化**：专注强化所选的终极天赋
4. **后期决策**：10级时慎重考虑轮回选择

### 终极天赋选择建议
- **新手玩家**：推荐终极追踪子弹，容错率高
- **进阶玩家**：推荐终极爆炸连锁，高上限高回报
- **高手玩家**：推荐终极穿透，技术要求高但效果强大
