## 📐 内容规范

### 版本号规范
- 遵循语义化版本控制 (Semantic Versioning)
- 格式：主版本号.次版本号.修订号 (如：1.0.0)
- 首个发布版本建议使用 0.1.0 或 1.0.0

### 版本类型说明
- **首个正式发布版本**: 项目的第一个稳定版本
- **重大更新**: 包含重要新功能或重大改进
- **小版本更新**: 包含少量新功能和改进
- **修复版本**: 主要修复bug和问题

### 内容长度控制
- 整个文档建议控制在 60-80 行以内
- 每个部分内容简洁明了，避免冗长描述
- 使用项目符号列表提高可读性

### 语言风格
- 使用简洁、专业的语言
- 适当使用表情符号增加亲和力
- 保持一致的术语和表达方式

## 🔧 技术要求

### 文件命名
- 格式：`RELEASE_v[版本号].md`
- 示例：`RELEASE_v0.1.0.md`

### 图标引用
- 使用相对路径引用项目图标
- 确保图标文件存在且可访问

### 链接规范
- 内部链接使用相对路径
- 外部链接确保可访问性

## ✅ 检查清单

发布前请确认以下内容：

- [ ] 版本号正确且符合规范
- [ ] 发布日期准确
- [ ] 所有链接可正常访问
- [ ] 系统要求信息准确
- [ ] 安装说明清晰可执行
- [ ] 已知问题已确认
- [ ] 未来计划合理可行
- [ ] 文档格式正确，无语法错误
- [ ] 内容长度适中，重点突出
